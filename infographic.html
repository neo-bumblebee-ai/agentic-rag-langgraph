Problem → Solution

❌ Traditional RAG Breaks Down

📄Retrieves documents regardless of whether they're actually relevant to the question
🎭Confidently generates wrong answers when context is empty or misleading (hallucination)
🔁Retries the exact same failing query — no learning from failure
🔍Only uses cosine similarity — misses keyword-specific matches
🧠Forgets every previous question — no conversational context
→

✅ Agentic RAG Solves It

🎯Document Grader scores every retrieved chunk — irrelevant ones are filtered out
🛡️Hallucination Checker tests if the answer is actually grounded in facts before delivering it
✏️Query Rewriter intelligently reformulates the question to improve retrieval accuracy
🔀Hybrid Search combines keyword matching + semantic similarity, fused by ranking math
💬Conversation Memory retains context across the session with automatic summarisation

Self-correction loop: If the Quality Gate fails — either because the answer is hallucinated or doesn't resolve the question — the system automatically rewrites the query and retries. This loop runs up to 3 times before returning the best available answer.
The 5 Specialized Agents
🧭
Router Agent
Decides whether to search internal documents or the live web. Routes the query to the right source before wasting retrieval cycles.
🔍
Retrieval Agent
Runs three retrieval methods in parallel — dense (semantic), sparse (keyword), and reranking — then combines results mathematically.
⚖️
Document Grader
Scores every retrieved chunk individually. Filters out irrelevant content before it can corrupt the generated answer.
🛡️
Hallucination Checker
Verifies the generated answer is actually supported by the retrieved documents — not invented. Triggers regeneration if it isn't.
✏️
Query Rewriter
When retrieval fails, rewrites the question to sharpen its semantic intent — improving the next retrieval pass automatically.
💬
Memory Manager
Maintains conversation context across turns. Automatically compresses history using the LLM when it grows too long.
The Retrieval Pipeline — Why 3 Methods Beat 1
1
Dense Search
Semantic similarity via AI embeddings (ChromaDB). Great for meaning-based matches.
+
2
Sparse Search
BM25 keyword scoring. Great for exact term matches that semantic search misses.
→
3
RRF Fusion
Reciprocal Rank Fusion mathematically merges both ranked lists without manual weights.
→
4
Re-Ranking
Cross-encoder reads query + doc together. 40-60% better precision than cosine alone.
→
5
Top-5 Chunks
Only the highest-confidence, most relevant context reaches the language model.
